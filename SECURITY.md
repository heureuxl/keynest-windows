# 安全策略

## 支持范围

维护者会认真对待与 **KeyNest for Windows 桌面应用**、**本机 HTTP 桥（`127.0.0.1:17373`）** 及 **本仓库内的 Chrome 扩展** 相关的安全报告，尤其是可导致以下情况的问题：

- 未授权读取或篡改本地保管库数据
- 绕过主密码/恢复密钥逻辑的错误
- 本机桥被非本机来源滥用（在默认可访问性假设下）

**不在此范围**：Chrome / Edge 对第三方网页的普遍同源策略限制、用户自行安装的恶意软件、物理接触设备后的离线攻击等。

## 如何报告

1. **不要**在公开 Issue 中披露可利用细节或粘贴 vault 文件、密钥。
2. 请优先通过 **GitHub Security Advisories**（仓库 → **Security** → **Advisories** → **Report a vulnerability**）提交。  
3. 若无法使用 Advisories，可发邮件至：**lq_17395@163.com**（主题建议含 `[Security] KeyNest`）。**请勿**在邮件正文中粘贴主密码、恢复密钥或完整 `vault.keynest` 文件内容。

请在报告中尽量包含：

- 受影响版本 / 提交范围
- 复现环境与步骤（操作系统、是否单文件发布、扩展版本）
- 是否为本地提权、仅本机影响或需网络条件（若有）

## 修复节奏

维护者会在合理时间内确认收到报告；严重问题优先处理。公开披露前会协调修复与发布说明。

## 用户侧建议

- 仅从**可信来源**获取发行的 `KeyNestForWin.exe`；核对 Release 附带的校验和（若提供）。
- **主密码与恢复密钥**仅保存在你自己控制的安全位置；不要通过聊天/邮件发送。
- 扩展与桌面端需**同时更新**时，请关注 GitHub Release 说明。
